# Padrão Geral para Agentes de IA

Este projeto deve ser desenvolvido de forma consistente, limpa e documentada.

## Regras principais

- Não criar padrões novos sem necessidade.
- Não duplicar código.
- Preferir arquivos pequenos e objetivos.
- Separar responsabilidades.
- Criar componentes reutilizáveis.
- Criar serviços pequenos.
- Não criar legado se o projeto não exigir.
- Não quebrar padrões já existentes.
- Não fazer mudanças grandes sem registrar decisão em `ai/DECISOES.md`.
- Sempre atualizar a memória do projeto (`ai/MEMORIA.md`).

## Antes de alterar código

O agente deve entender:
- O objetivo da tarefa.
- O padrão atual do projeto.
- A estrutura de pastas.
- As decisões já tomadas (`ai/DECISOES.md`).
- As pendências conhecidas (`ai/PENDENCIAS.md`).

## Ao finalizar

O agente deve informar:
1. O que foi feito
2. Arquivos alterados
3. Testes executados
4. Pendências
5. Riscos
6. Próximo passo recomendado
